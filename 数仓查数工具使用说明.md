# 数仓查数工具使用说明

`superset-query` 是命令行查数工具，通过 Superset 连接 DLC 数仓。在终端跑一行命令即可查数据、看表结构、导出 CSV，无需打开 Superset 网页，也无需安装 Python 依赖。

---

## 快速开始

### 1. 获取工具

下载 `superset-query` 二进制，放到项目目录：

```
data/dist/superset-query
```

> 下载地址：[企业微信网盘](https://drive.weixin.qq.com/s?k=AFcAOAfiAAYAR652aDAXkAIwYRALY)

工具可**单独使用**，不依赖 skill 或 Python。

### 2. 设置别名（推荐）

```bash
alias sq='./dist/superset-query'
```

### 3. 首次使用

在 `data/` 目录下直接运行任意查数命令，工具会引导完成配置：

```
==================================================
Superset CLI - First Time Setup
==================================================
Superset username: 你的账号
Superset password: 你的密码

正在验证账号并获取可用数据源...

可用数据源（请选择默认数据源）
--------------------------------------------------
  [1] ID=4: DataLakeCatalog (doris)
  ...

选择默认数据源 [1-N]，或直接输入 ID（回车使用默认）: 4
```

首次配置会保存到 `~/.config/superset-cli/`，后续自动登录，无需重复输入。

| 情况 | 处理方式 |
|------|----------|
| 账号密码错误 | 提示重试，最多 3 次 |
| 网络不通 | 立即退出，检查网络或 VPN |
| 登录成功但无数据源权限 | 联系管理员开通权限 |

## 常用命令

### 查数据

```bash
# 看样例（注意：分区表必须带 dt）
sq --sql "SELECT * FROM DataLakeCatalog.ods_log.user_benefit_plan WHERE dt='2026-06-24' LIMIT 10"

# 导出 CSV（推荐）
sq --sql "SELECT user_id, pay_time, total_fee FROM DataLakeCatalog.dwd_db.dwd_order_lifecycle WHERE dt='2026-06-24' AND pay_status=1 LIMIT 100" --output-file 付费用户.csv

# 从 SQL 文件执行
sq --sql-file data/sql/monthly_limit_99_popup_conversion.sql --output-file data/out/结果.csv
```

### 看表结构

```bash
# 字段名、类型、注释
sq --describe DataLakeCatalog.ods_log.user_benefit_plan

# 建表语句
sq --show-create DataLakeCatalog.ods_log.user_benefit_plan
```

### 探索数仓

```bash
# 列出所有数据源
sq --list-datasource

# 列出某数据源下的数据库
sq --list-databases --datasource-id 4

# 列出某库下的表
sq --list-tables --datasource-id 4 --database ods_log
```

## 重要规则：分区表必须带 dt

`DataLakeCatalog` 下的表大多按 `dt`（日期）分区。**所有查询都必须带 dt 条件**，否则全表扫描，既慢也可能被限流。

```sql
-- ✅ 正确
SELECT * FROM DataLakeCatalog.ods_log.user_benefit_plan
WHERE dt >= '2026-06-23' AND dt <= '2026-06-24'
LIMIT 10;

-- ❌ 错误
SELECT * FROM DataLakeCatalog.ods_log.user_benefit_plan LIMIT 10;
```

不确定查哪几天时，先问清楚统计日期，或查最近分区：

```bash
sq --sql "SELECT MAX(dt) AS latest_dt FROM DataLakeCatalog.ods_log.user_benefit_plan WHERE dt >= '2026-01-01'"
```

## 参数速查

| 你想做什么 | 命令 |
|-----------|------|
| 查数据 | `--sql "SELECT ..."` |
| 从文件读 SQL | `--sql-file 查询.sql` |
| 结果存 CSV | `--output-file 结果.csv` |
| 结果存 JSON | `--output-file 结果.json --format json` |
| 指定输出目录 | `--output-dir ./data/out` |
| 看表字段 | `--describe 库.表` |
| 看建表语句 | `--show-create 库.表` |
| 列出数据源 | `--list-datasource` |
| 列出数据库 | `--list-databases --datasource-id 4` |
| 列出表 | `--list-tables --database 库名` |
| 重复查询加速 | `--cache` |
| 查看查询历史 | `--history` |
| 清空缓存 | `--clear-cache` |
| 清空历史 | `--clear-history` |
| 强制重新登录 | `--force-login` |
| 设置默认数据源 | `--set-default-datasource 4` |
| 查看全部参数 | `--help` |

## 常用库表

| 层级 | 路径示例 | 说明 |
|------|----------|------|
| ODS 日志 | `DataLakeCatalog.ods_log.*` | 原始行为日志 |
| DWD 明细 | `DataLakeCatalog.dwd_db.*` | 订单等明细宽表 |

示例：

- `ods_log.user_benefit_plan` — 用户福利计划弹窗日志
- `ods_log.random_disconnect_limit` — 随机断联策略日志
- `dwd_db.dwd_order_lifecycle` — 订单生命周期

表名和字段含义不确定时，**先 `--describe` 或问 AI，不要猜**。

## 常见问题

**Q：第一次要输入什么？**

Superset 账号密码。验证通过后会列出可用数据源，选一个作为默认（一般选 `DataLakeCatalog`，ID=4）。

**Q：后续还要输密码吗？**

不用。凭据和登录状态保存在 `~/.config/superset-cli/`，自动复用。

**Q：报错「认证失败」或登录异常？**

```bash
rm -rf ~/.config/superset-cli/
sq --sql "SELECT 1"   # 重新走首次设置
```

**Q：命令找不到文件？**

确认当前目录是 `data/`（`pwd` 检查），或使用完整路径 `./dist/superset-query`。

**Q：查出来的数不对？**

1. 检查 `dt` 日期范围是否正确  
2. 确认指标口径（如用 `total_fee` 还是 `actual_price`、是否剔除退款）  
3. 让 AI 重新探表验证，不要凭印象改 SQL

**Q：查询很慢？**

检查 SQL 是否缺少 `dt` 分区条件；大结果集可加 `LIMIT`，或用 `--output-file` 导出后本地分析。
